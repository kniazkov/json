enum JsonParsingMode

Specifies the parsing mode of the JSON document.

In total, there are three parsing modes:

JsonParsingMode.STRICT

The most brutal regime. Only the syntax defined by the ECMA-404 standard ("The JSON Data Interchange Standard") is allowed.
Parsing syntax constructs not defined by this standard will raise an exception.
All competing parsers support this mode.

JsonParsingMode.JSON5

Supports extended syntax. Such JSON documents are easier for humans to read and modify. You'll love this.
For example, you don't have to put the key in quotes: 

{
    name: "John",
    age: 17
}

JsonParsingMode.EXTENDED

Do you make mistakes? I think I do. And that includes if you are editing a JSON document that is large enough to get confused.
You might, say, accidentally forget to drink your morning coffee put a comma between two objects, like this: 

[
    {
        name: "John",
        age: 17
    }
    {
        name: "Anna",
        age: 16
    }
]
And of course, no other parser will bother with this. It would just throw an exception.
But not this one. In this mode, we will still try to parse this somehow (and forgive the user who has to edit a large document and hasn't had his morning coffee).
After all, we all make mistakes.
This is the default mode unless another mode is specified.
